S

Il Gateway inoltra le richieste ai microservizi via HTTP su rete Docker privata. I servizi possono chiamarsi tra loro per operazioni che richiedono risposta immediata (es. scheduling verifica disponibilità operatore su auth-service).

gateway/src/index.js — Bootstrap 
import Fastify from 'fastify'
import cors from '@fastify/cors'
import jwt from '@fastify/jwt'
import rateLimit from '@fastify/rate-limit'
import websocket from '@fastify/websocket'
import proxy from '@fastify/http-proxy'
import { jwtMiddleware } from './middleware/jwt.js'
import { rbacMiddleware } from './middleware/rbac.js'
import { auditMiddleware } from './middleware/audit.js'
import { registerRoutes } from './routes/index.js'

const app = Fastify({ logger: true, trustProxy: true })

// Plugins
await app.register(cors, {
  origin: process.env.ALLOWED_ORIGINS.split(','),
  credentials: true
})

await app.register(rateLimit, {
  max: 100,
  timeWindow: '1 minute',
  keyGenerator: (req) => req.headers['x-forwarded-for'] || req.ip
})

await app.register(jwt, {
  secret: process.env.JWT_ACCESS_SECRET,
  sign: { expiresIn: '15m' }
})

await app.register(websocket)

// Hooks globali
app.addHook('onRequest', jwtMiddleware)
app.addHook('onRequest', rbacMiddleware)
app.addHook('onResponse', auditMiddleware)

await registerRoutes(app)

await app.listen({ port: 3000, host: '0.0.0.0' })
console.log('Gateway running on :3000')
middleware/jwt.js — Verifica token + route pubbliche 
const PUBLIC_ROUTES = [
  { method: 'POST', path: '/api/v1/auth/login' },
  { method: 'POST', path: '/api/v1/auth/refresh' },
  { method: 'POST', path: '/api/v1/wb/reports' },  // whistleblowing anonimo
]

export async function jwtMiddleware(request, reply) {
  const isPublic = PUBLIC_ROUTES.some(r =>
    r.method === request.method && request.url.startsWith(r.path)
  )
  if (isPublic) return

  try {
    await request.jwtVerify()
    // Payload decodificato disponibile in request.user
    // { sub: userId, role: 'operator', iat, exp }
  } catch (err) {
    return reply.code(401).send({
      error: 'UNAUTHORIZED',
      message: 'Token non valido o scaduto'
    })
  }
}

Flusso autenticazione — OAuth2 + JWT dual-token
1
Login → POST /api/v1/auth/login
Client invia email + password. Il servizio verifica bcrypt hash, controlla is_active, aggiorna last_login_at.
2
Generazione token doppio
Access Token: JWT firmato HS256, scadenza 15 minuti, payload: { sub, role, jti }
Refresh Token: stringa opaca 256-bit, TTL 7 giorni, salvato in Redis con chiave rt:{userId}:{jti}
access_token → header Authorization: Bearer <token>
refresh_token → cookie HttpOnly, Secure, SameSite=Strict
3
Refresh → POST /api/v1/auth/refresh
Client invia cookie refresh. Server verifica in Redis, emette nuovo access token, ruota il refresh token (vecchio invalidato).
4
Logout → POST /api/v1/auth/logout
Cancella la chiave Redis del refresh token. Tutti i device dell'utente vengono scollegati se si usa logout-all.
5
MFA (opzionale per admin/coordinator)
Dopo login con credenziali corrette, se mfa_secret è presente, il server restituisce uno stato intermedio che richiede TOTP (Google Authenticator / RFC 6238).
POST /api/v1/auth/mfa/verify body: { totp_code: "123456" }

Meccanismo Proof-of-Presence — Come funziona
1
L'operatore arriva dal beneficiario
L'app Flutter acquisisce GPS coordinates, timestamp device, e device_id univoco dell'installazione.
2
L'app costruisce il payload firmato
payload = { appointment_id, operator_id, lat, lng, timestamp }
signature = HMAC-SHA256(payload, device_secret)
Il device_secret viene assegnato al momento dell'onboarding dell'operatore e salvato in modo sicuro nel KeyStore Android / Secure Enclave iOS.
3
Il server verifica
Il presenze-service riceve il payload, ricalcola la firma lato server con il device_secret registrato nel DB, confronta. Se corrispondono: is_verified = true. Verifica anche la coerenza geografica con l'indirizzo del beneficiario (geofence ±200m).
4
Fallback offline
Se l'operatore è senza connessione, l'app salva il record in SQLite locale e lo sincronizza alla prima connessione disponibile, entro un massimo di 2 ore (configurabile).
